### TASK-294: Preserve Closed Task Bodies in Quarterly Archive Shards
**Priority**: P1 (High)
**Estimate**: 3-5 hours

The current compact-ledger workflow removes full task definitions from the live
backlog when a task closes, which keeps active context small but also makes
closed task bodies hard to inspect unless they happened to be captured in an
older dated archive snapshot. Update task-closing workflow and archive-aware
lookup so every closed task preserves its full markdown block in a quarterly
archive shard such as `archive/closed_tasks/2026-Q1.md`, while keeping live
agent/default CLI reads strictly archive-free unless explicitly requested.

**Files**: `tasks/BACKLOG.md`, `tasks/COMPLETED.md`, `archive/closed_tasks/`, `src/horadus_cli/task_repo.py`, `src/horadus_cli/task_commands.py`, `src/core/docs_freshness.py`, `tests/`, `docs/AGENT_RUNBOOK.md`, `AGENTS.md`, `README.md`

**Acceptance Criteria**:
- [ ] Closing a task removes its full block from live `tasks/BACKLOG.md`, adds the compact completion entry to `tasks/COMPLETED.md`, and appends the full task block to the appropriate quarterly archive shard (for example `archive/closed_tasks/2026-Q1.md`)
- [ ] Archive shard format is stable and parseable enough for archive-aware CLI lookup without forcing agents to read archived task bodies by default
- [ ] `horadus tasks show`, `search`, and `context-pack` continue to stay live-only by default but can resolve closed-task bodies from `archive/closed_tasks/*.md` when archive access is explicitly requested
- [ ] Agent guidance/documentation state clearly that `archive/closed_tasks/` is non-authoritative for normal implementation flow and should only be consulted on explicit archive-aware requests
- [ ] Regression tests cover at least one close/archive path and one archive-aware lookup path using the new quarterly archive shard layout
